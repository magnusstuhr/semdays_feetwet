exists-P=false where P is a chain

The query specification exists-P=false where P is a chain of at least two elements does not work.

cache

Elda caches the results of queries so that they may be re-served quickly. When the cache gets "too full", it is reset. "Too full" by default is measured by the number of triples in the cache. See the more detailed documentation [upcoming] on how to control the cache.
